Do not add dashboard cards or secondary information to the HomePage. The homepage's job is scene management and subway status — those are the primary user actions. Everything else (device insights, battery, environment, energy) belongs elsewhere.

**Why:** The user explicitly rejected building dashboard features on the homepage. Cluttering it with secondary information degrades the core experience of managing scenes and checking the subway.

**How to apply:** When planning new features, always ask "does this belong on the homepage?" — if it's not scene management or commute-related, it goes on a dedicated page.
